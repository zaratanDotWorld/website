# CRO reports

Monthly conversion-rate-optimization reports generated by the CRO Agent (`.github/workflows/cro-agent.yml`).
Each `report-<YYYY-MM-DD>.md` file contains the GA4 data window for that run plus Claude's analysis and prioritized recommendations.

## Setup

The workflow needs two repo secrets and one Google Cloud service account bound to the GA4 property as a Viewer.

### Service account

1. In Google Cloud Console, create a service account in any project (we use `cro-agent@zaratan.iam.gserviceaccount.com`).
2. Generate a JSON key for it and store the full JSON as the `GA4_SERVICE_ACCOUNT_KEY` repo secret.
3. Bind the service account to the GA4 property as Viewer.

### GA4 binding workaround

The GA4 admin UI currently rejects `*.iam.gserviceaccount.com` emails outright (see [Google support thread](https://support.google.com/analytics/thread/427838065)).
Use the [Analytics Admin API Explorer](https://developers.google.com/analytics/devguides/config/admin/v1/rest/v1alpha/properties.accessBindings/create) instead:

- `parent`: `properties/<property_id>` (e.g. `properties/411790166` — find the numeric ID in GA Admin → Property details, *not* the `G-...` measurement ID)
- Request body:
  ```json
  {
    "user": "<service-account>@<project>.iam.gserviceaccount.com",
    "roles": ["predefinedRoles/viewer"]
  }
  ```
- Click **Execute**, sign in with the Google account that has Admin access on the property. A 200 means the binding stuck. Verify in GA Admin → Property Access Management.

### Other requirements

- `CLAUDE_CODE_OAUTH_TOKEN` repo secret (generate with `claude setup-token` locally).
- "Allow GitHub Actions to create and approve pull requests" enabled at org or repo level — required for the action to open the report PR.
