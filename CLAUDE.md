# CLAUDE.md

Zaratan marketing site.
Next.js 15 (Pages Router), React 18, Bootstrap, deployed via AWS Amplify.

## Stack

- Package manager: `pnpm` (see `engines` in `package.json`, Node >=22).
- Pages live under `pages/` as `.jsx`.
- Shared UI in `components/`, styles in `styles/`, helpers in `utils/`.
- CTA event names are defined in `utils/track.js`.

## Product context

Living product docs sit under `spec/`, organized as two businesses (coliving, Chore Wheel), each with a product spec and a go-to-market playbook, under one strategy doc:

- `spec/STRATEGY.md` — overall Zaratan strategy (two businesses, growth priorities)
- `spec/CO-PROD.md` — coliving product spec
- `spec/CO-GTM.md` — coliving go-to-market (positioning, pricing, funnel, competitors)
- `spec/CO-TESTIMONIALS.md` — verbatim resident quotes, raw material for listings and marketing copy
- `spec/CW-PROD.md` — Chore Wheel product spec
- `spec/CW-GTM.md` — Chore Wheel go-to-market (pitch, pilot playbook)

## Analytics

Everything related to traffic and conversion analysis lives under `analytics/`:

- `analytics/cro/` — monthly CRO reports generated by `.github/workflows/cro-agent.yml`. Setup notes are in `analytics/cro/README.md`.
- `analytics/gsc/` — Google Search Console snapshots, named `report-<MM-DD-YYYY>.md`.
- `analytics/pricing/` — rental-market snapshots, named `report-YYYY-MM-DD.md`. At session start, if the most recent report is more than ~15 days old, remind me to refresh it — I update these myself, so just nudge rather than trying to regenerate them.

The CRO agent runs `scripts/cro-agent/run.mjs` to pull GA4 data, then Claude writes the report into `analytics/cro/`.

## Conventions

- Markdown: one sentence per line.
- Commits: conventional prefixes (`feat:`, `fix:`, `docs:`, `chore:`, `refactor:`).
- Don't commit or open PRs without explicit go-ahead.
